# Methodology

## Task representation

A task is represented as a causal dynamical system:

```text
dx/dt = f_A(x, u)
```

where `A` is a generated causal interaction matrix and `u` is an intervention vector.

## Selection criterion

Tasks are selected by a combination of:

- novelty,
- solvability,
- learnability,
- complexity.

A task is preferred when it is not trivial, not impossible, and structurally different from archived tasks.

## Archive mechanism

Accepted tasks are stored in an archive. Their descendants are generated by mutation, producing an open-ended search process.

## Transfer matrix

The transfer matrix estimates similarity between archived tasks. It is a proxy for whether solutions or world models may transfer across tasks.
